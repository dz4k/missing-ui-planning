## Framework-free
No. Material UI is built specifically for React.
> "Material UI is a library of React components." (https://mui.com/material-ui/getting-started/)

## Framework-independent
No. While MUI Base offers unstyled components, they are still React components.
> "MUI Base is a library of headless ('unstyled') React components and low-level hooks." (https://mui.com/base-ui/getting-started/)

## Headless
Yes, through MUI Base.
> "MUI Base... offers unstyled components as an alternative to the fully styled Material UI library. It provides the foundational logic of Material UI components without any imposed styles." (https://mui.com/base-ui/introduction/)

## Has styles
Yes. The core Material UI library is a styled implementation of Material Design.
> "Material UI is a comprehensive library of components that features our implementation of Google's Material Design." (https://mui.com/material-ui/introduction/)

## Styleable
Config-styleable (Material UI) / Fully styleable (MUI Base).
> "You gain complete control over your app's CSS and accessibility features... if you prefer Emotion, Tailwind CSS, plain CSS, or any other styling engine, they are available too!" (https://mui.com/base-ui/getting-started/)

## Classless
No. It requires specific component usage and often CSS-in-JS or specific class names.

## Buildless
No. It requires a build step and React environment.
