## Framework-free
Yes.
> "These components are based on W3C Web Component standards, making them inherently framework-agnostic." (https://www.fast.design/docs/introduction/)

## Framework-independent
Yes.
> "You can use them directly with plain HTML, CSS, and JavaScript, without requiring complex frameworks like React or Angular." (https://www.fast.design/docs/concepts/why-fast/)

## Headless
Yes, through FAST Foundation.
> "FAST provides 'Foundation Elements for Styling' and a configurable design system called FAST Frame... to enable extensive customization." (https://www.fast.design/docs/design-systems/overview/)

## Has styles
Yes, via implementations like Fluent UI.

## Styleable
Unstyleable (FAST Foundation) / Config-styleable (Fluent UI).
> "FAST Frame... utilizes Design Tokens to enable extensive customization of visual design, including color, typography, and spacing." (https://www.fast.design/docs/design-systems/fast-frame/)

## Classless
No.

## Buildless
Yes.
> "For a buildless approach, you can directly import Fluent UI Web Components into your HTML files via a Content Delivery Network (CDN)." (https://www.fast.design/docs/getting-started/html-and-javascript/)
